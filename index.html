Why Kind-ling

Agent adoption is a configuration decision, not a marketing funnel.

Five mechanisms determine how AI agents discover, evaluate, and adopt services: MCP tool descriptions, A2A Agent Cards, x402 settlement, context window content, and on-chain transaction history.

The connective layer — reputation scoring, task routing, description optimization — is unoccupied. Kind-ling occupies it with tools that each provide standalone value and, when composed, create reinforcing loops across all five mechanisms.

Principles

Infrastructure, not hype. Precision, not jargon.

Open-source, bypassable

Every tool runs standalone. No lock-in. Fork it, host it, bypass it entirely. Works because it's useful.

Methodology before scores

Scoring methodology is published before any score is emitted. Anyone can reproduce from on-chain data.

First-party disclosed

PUC operates services that participate in Kind-ling. Labeled is_first_party: true throughout. No scoring advantages.

Referral ≠ routing

Split size has zero effect on ranking or routing position. Hard rule, enforced in code.

Economics on-chain

Every 402 response discloses referral economics before payment. All splits auditable by anyone.

Honest about trade-offs

We say what we build and what we don't. Scope boundaries are explicit and public.
